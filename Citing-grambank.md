We prefer that if you use grambank data you cite both the release paper _and_ the dataset release (for the specific version you used). Citing the dataset version ensures replicability. Below are APA citations and bibTeX-entries for each, which can be imported into Zotero etc. 

It is not possible to cite individual Grambank features. However, each feature has one or two experts who are in charge of it, these are known as "patrons". You can find who is patron of what in the ParameterTable in the CLDF-release and at the bottom of each feature wiki article. If you can, pay attention to the patron(s) for each feature and mention them if there is anything specific you want to highlight about their work.

We prefer that you include all names if possible. [See section below in regards to non-ASCII characters in TeX.](https://github.com/grambank/grambank/wiki/Citing-grambank#character-encoding)

For more practical guides for using Grambank data, go [here](https://github.com/grambank/grambank/wiki#2-practical-information-for-users-and-collaborators). For a list of publications from the project, please go [here](https://github.com/grambank/grambank/wiki/Grambank-publications).

### Character encoding


Guide for non-ASCII characters in TeX notation:
```
{\aa} = å
{\'a} = á
{\"o} = ö
{\'e} = é
{\"u} = ü
{\^o} = ô
{\ss} = ß
$\breve{g}$ = ğ
```
If you write in TeX, we recommend that you use XeLaTeX for compilation to ensure the representation of Chinese characters.
